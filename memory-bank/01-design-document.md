## 1. 产品概述

### 1.1 产品定义

专属工具箱是一个面向个人用户的工具集合平台，允许用户将常用的在线网页工具和离线工具包分类收录，打造个人专属的工具箱。

### 1.2 解决问题

- 解决常见工具箱网站工具内容过多、难以快速定位的问题
- 解决无法添加新发现工具的问题
- 满足用户对离线工具包的管理需求
- 无需登录即可使用，保护用户隐私

### 1.3 产品价值

为用户提供一个个性化、高效的工具管理平台，让用户能够快速访问常用的在线工具和离线工具包。

## 2. 范围与边界

### 2.1 功能范围

#### 2.1.1 核心功能（必须实现）

| 功能模块 | 功能项 | 说明 |
|----------|--------|------|
| 工具展示 | 卡片式布局 | 以简洁卡片式布局展示工具 |
| 工具展示 | 加载预设工具 | 从 `prese-tool.md` 加载预设工具名单 |
| 工具展示 | 分类筛选 | 支持按全部、我的收藏、预设分类和自定义分类筛选 |
| 工具展示 | 搜索功能 | 支持关键字搜索工具名称和描述 |
| 工具访问 | 在线工具访问 | 点击卡片新窗口打开工具URL |
| 工具访问 | 离线工具访问 | 将工具箱目录下的工具包复制到用户指定路径 |
| 工具添加 | 在线工具添加 | 手动输入工具网址、工具名称、工具描述，选择工具所属分类完成添加 |
| 工具添加 | 离线工具添加 | 手动上传工具包文件、输入工具名称、工具描述，选择工具所属分类完成添加 |
| 工具添加 | 手动图标上传 | 支持用户上传自定义图标 |
| 工具编辑 | 信息修改 | 修改工具名称、描述、图标、分类 |
| 工具删除 | 单个删除 | 删除指定的工具 |
| 工具收藏 | 收藏/取消收藏 | 标记/取消标记工具为收藏 |
| 分类管理 | 预设分类 | 使用系统预设的分类 |
| 分类管理 | 自定义分类 | 创建、编辑、删除自定义分类 |
| 数据导出 | 全部导出 | 导出全部工具数据为JSON文件 |
| 数据导出 | 部分导出 | 仅导出在线工具 |
| 数据导入 | 数据导入 | 从JSON文件导入工具数据 |

#### 2.1.2 非功能范围（明确不实现）

| 功能模块 | 说明 |
|----------|------|
| 用户系统 | 不需要用户注册、登录 |
| 云端同步 | 不支持多设备数据同步 |
| 预置工具包 | 平台不提供离线工具包下载 |
| 社区分享 | 不支持工具集合分享 |
| 批量操作 | 不支持批量添加/删除/编辑 |
| 多语言 | 不支持语言切换 |

### 2.2 技术边界

| 边界项 | 约束条件 |
|--------|----------|
| 存储方案 | IndexedDB 本地存储 + 本地文件系统（离线工具包） |
| 离线存储位置 | 离线工具包存储于「工具箱软件目录/offline-tools/」下，按工具 ID 分目录存放 |
| 存储容量 | 单个工具图标 base64 不超过 100KB，单个离线工具包不超过 50MB |
| 工具数量 | 不限制工具数量（受浏览器存储限制） |
| 分类数量 | 不限制分类数量 |
| 浏览器支持 | Chrome、Firefox、Safari、Edge 最新两个版本 |
| 设备适配 | 桌面端优先，移动端适配 |

### 2.3 数据边界

| 数据类型 | 存储位置 | 可迁移性 |
|----------|----------|----------|
| 工具数据 | IndexedDB | 可通过导出导入迁移 |
| 分类数据 | IndexedDB | 可通过导出导入迁移 |
| 离线工具包 | 本地文件系统（工具箱软件目录） | 随导出文件一并导出 |
| 用户偏好 | localStorage | 不可迁移（浏览器本地） |
| 浏览器缓存 | 内存/缓存 | 不可迁移 |

## 3. 目标用户

个人用户，需要一个专属的工具箱来管理和快速访问常用在线工具和离线工具包的用户。

### 3.1 用户特征

- 需要管理多个常用在线工具
- 有离线工具包管理需求
- 注重个人隐私，不希望注册登录
- 注重数据可控性，希望离线工具包不易丢失

### 3.2 使用场景

- 日常工作中快速访问常用工具网站
- 整理和归类自己积累的工具资源
- 换设备时迁移工具配置
- 管理本地安装的离线工具

## 4. 用户旅程

### 4.1 工具发现与访问

#### 场景1：首次访问工具箱

```
用户操作 → 系统响应

1. 用户打开工具箱网页
2. 系统从 prese-tool.md 加载预设工具名单
3. 系统显示首页，展示预设的示例工具卡片
4. 顶部显示搜索框、导出/导入按钮
5. 分类标签栏显示"全部"、"我的收藏"及预设分类和自定义分类
```

#### 场景2：按分类浏览工具

```
用户操作 → 系统响应

1. 用户点击分类标签（如"开发工具"）
2. 系统筛选显示该分类下的工具卡片
3. 分类标签高亮显示当前选中分类
4. 工具卡片数量相应更新
```

#### 场景3：使用搜索功能

```
用户操作 → 系统响应

1. 用户在搜索框输入关键字
2. 系统实时过滤，显示匹配工具名称或描述的工具卡片
3. 清空搜索框后恢复显示全部工具
```

#### 场景4：访问在线工具

```
用户操作 → 系统响应

1. 用户点击在线工具卡片
2. 系统在新浏览器标签页打开工具URL
3. 当前工具箱页面保持不变
```

#### 场景5：访问离线工具包

```
用户操作 → 系统响应

1. 用户点击离线工具包卡片
2. 系统检查工具箱软件目录下的工具包文件是否存在
   - 文件存在：弹出文件保存对话框，让用户选择保存路径，将工具包复制到该路径
   - 文件不存在：提示"工具包文件丢失，是否重新上传？"并删除此工具记录
```

### 4.2 添加工具

#### 场景6：添加在线工具

```
用户操作 → 系统响应

1. 用户点击"+"添加工具按钮
2. 系统打开添加工具对话框，默认为在线工具
3. 用户输入工具网址（必填）
4. 用户输入工具名称（必填）
5. 用户上传图标或选择图标（可选）
6. 用户输入工具描述（可选）
7. 用户选择分类（必填）
8. 用户点击"保存"
9. 系统验证数据（URL格式、必填项）
   - 验证通过：保存到IndexedDB，关闭对话框，刷新列表
   - 验证失败：显示错误提示
10. 系统显示成功提示"工具添加成功"
```

#### 场景7：添加离线工具包

```
用户操作 → 系统响应

1. 用户点击"+"添加工具按钮
2. 系统打开添加工具对话框
3. 用户切换到"离线工具包"选项卡
4. 用户上传工具包文件（必填）
5. 系统将工具包保存到「工具箱软件目录/offline-tools/{工具ID}/」下
6. 用户输入工具包名称（必填）
7. 用户上传或选择图标（可选）
8. 用户输入描述（可选）
9. 用户选择分类（必填）
10. 用户点击"保存"
11. 系统验证数据（必填项、文件包大小不超过50MB）
    - 验证通过：保存记录到IndexedDB，关闭对话框，刷新列表
    - 验证失败：显示错误提示
12. 系统显示成功提示"工具添加成功"
```

### 4.3 管理工具

#### 场景8：编辑工具信息

```
用户操作 → 系统响应

1. 用户悬停工具卡片，显示编辑按钮
2. 用户点击编辑按钮
3. 系统打开编辑工具对话框，预填充当前数据
4. 用户修改信息
5. 用户点击"保存"
6. 系统更新IndexedDB中的数据
7. 系统显示成功提示"工具更新成功"
```

#### 场景9：删除工具

```
用户操作 → 系统响应

1. 用户悬停工具卡片，显示删除按钮
2. 用户点击删除按钮
3. 系统显示确认对话框"确定要删除该工具吗？"
4. 用户点击"确定"
5. 系统从IndexedDB删除该工具记录
6. 如果是离线工具，同时删除「工具箱软件目录/offline-tools/{工具ID}/」下的文件
7. 系统显示成功提示"工具已删除"
8. 工具卡片从列表中移除
```

#### 场景10：收藏工具

```
用户操作 → 系统响应

1. 用户点击工具卡片的收藏按钮（星标图标）
2. 系统更新工具的isFavorite状态为true
3. 收藏按钮变为已收藏状态（金色/高亮）
4. 系统显示成功提示"已添加到收藏"
```

#### 场景11：取消收藏

```
用户操作 → 系统响应

1. 用户在收藏列表中点击已收藏工具的星标按钮
2. 系统更新工具的isFavorite状态为false
3. 收藏按钮恢复默认状态
4. 系统显示成功提示"已取消收藏"
5. 如果用户在"我的收藏"页面，工具卡片从列表中移除
```

### 4.4 管理分类

#### 场景12：创建自定义分类

```
用户操作 → 系统响应

1. 用户在分类管理界面点击"新建分类"
2. 系统显示新建分类对话框
3. 用户输入分类名称
4. 用户点击"保存"
5. 系统创建新分类，保存到IndexedDB
6. 新分类出现在分类列表中
```

#### 场景13：删除自定义分类

```
用户操作 → 系统响应

1. 用户在分类管理界面点击删除按钮
2. 系统显示确认对话框"删除分类后，该分类下的工具将移至'未分类'，确定删除？"
3. 用户点击"确定"
4. 系统删除分类，将该分类下的工具移至"未分类"
5. 系统显示成功提示"分类已删除"
```

### 4.5 数据导出导入

#### 场景14：导出全部数据

```
用户操作 → 系统响应

1. 用户点击"导出数据"按钮
2. 系统显示导出选项对话框
3. 用户选择"导出全部工具"
4. 用户点击"导出"
5. 系统从IndexedDB读取全部工具和分类数据
6. 系统从「工具箱软件目录/offline-tools/」读取所有离线工具包文件
7. 系统生成JSON格式的导出文件（包含工具数据、分类数据、离线工具包base64）
8. 浏览器下载文件，文件名为"toolbox-export-{日期}.json"
```

#### 场景15：仅导出在线工具

```
用户操作 → 系统响应

1. 用户点击"导出数据"按钮
2. 系统显示导出选项对话框
3. 用户选择"仅导出在线工具"
4. 用户点击"导出"
5. 系统过滤出isOffline=false的工具
6. 系统生成JSON格式的导出文件
7. 浏览器下载文件
```

#### 场景16：导入数据

```
用户操作 → 系统响应

1. 用户点击"导入数据"按钮
2. 系统打开文件选择器
3. 用户选择JSON导出文件
4. 系统读取并解析JSON文件
5. 系统验证文件格式
   - 格式正确：显示导入预览（工具数量、分类数量、离线工具包数量）
   - 格式错误：显示错误提示"文件格式不正确"
6. 系统检查重复工具
   - 对于在线工具：以「工具网址+工具名称」组合作为重复校验依据
   - 对于离线工具：以「工具包名称+工具名称」组合作为重复校验依据
   - 发现重复：显示重复工具列表，让用户选择"跳过"或"覆盖"
7. 用户点击"确认导入"
8. 系统执行数据导入
   - 离线工具包保存到「工具箱软件目录/offline-tools/{工具ID}/」
9. 系统显示成功提示"成功导入X个工具，X个分类"
```

## 6. 边界情况

### 6.1 工具添加边界情况

| 边界情况 | 处理方式 |
|----------|----------|
| URL格式不正确 | 提示"请输入有效的URL地址" |
| URL无法访问 | 仍可保存，但提示"无法获取网站信息，请手动填写" |
| 工具名称为空 | 提示"工具名称不能为空" |
| 离线工具包未选择 | 提示"请选择工具包文件" |
| 图标超过100KB | 提示"图标文件过大，请选择小于100KB的图片" |
| 离线工具包超过50MB | 提示"工具包文件过大，请选择小于50MB的文件" |
| 分类名称重复（创建分类时） | 提示"分类名称已存在" |
| 同URL同名称的工具 | 提示"该工具已存在，是否继续添加？" |
| IndexedDB存储已满 | 提示"存储空间不足，请删除一些工具或导出数据" |
| 浏览器隐私模式 | 弹窗警告"当前为隐私模式，数据将无法保存" |

### 6.2 工具访问边界情况

| 边界情况 | 处理方式 |
|----------|----------|
| 在线工具URL为空 | 提示"工具URL为空，无法访问" |
| 在线工具URL格式错误 | 提示"工具URL格式错误" |
| 离线工具包文件不存在 | 提示"工具包文件丢失，是否重新上传？"并删除此工具记录 |
| 离线工具包文件被移动 | 同上，按"文件不存在"处理 |

### 6.3 搜索边界情况

| 边界情况 | 处理方式 |
|----------|----------|
| 搜索关键字为空 | 显示全部工具 |
| 搜索关键字全空格 | 显示全部工具 |
| 搜索结果为空 | 显示"未找到匹配的工具" |
| 搜索关键字过长 | 超过50字符时截断输入 |

### 6.4 数据导入边界情况

| 边界情况 | 处理方式 |
|----------|----------|
| 文件格式错误 | 提示"文件格式不正确，请选择有效的导出文件" |
| 文件内容损坏 | 提示"文件内容损坏，无法导入" |
| 导入数据为空 | 提示"导入文件中没有数据" |
| 版本不兼容 | 提示"文件版本不兼容，当前支持v1格式" |

### 6.5 存储边界情况

| 边界情况 | 处理方式 |
|----------|----------|
| IndexedDB存储已满 | 提示"存储空间不足，请删除一些工具或导出数据" |
| 浏览器隐私模式 | 弹窗警告"当前为隐私模式，数据将无法保存" |
| IndexedDB不可用 | 提示"您的浏览器不支持数据存储，请更换浏览器" |

## 7. 数据模型

### 7.1 工具模型（Tool）

```typescript
interface Tool {
  id: string;                    // 工具唯一标识（UUID）
  name: string;                  // 工具名称（最长50字符）
  url: string;                   // 工具URL（仅在线工具）
  icon: string;                  // 工具图标（base64或URL）
  description: string;           // 工具描述（最长200字符）
  categoryId: string;            // 分类ID
  isFavorite: boolean;           // 是否收藏
  isOffline: boolean;           // 是否为离线工具
  offlineFileName: string;      // 离线工具包文件名（仅离线工具）
  createdAt: number;             // 创建时间戳
  updatedAt: number;             // 更新时间戳
}
```

### 7.5 离线工具包存储结构

```
工具箱软件目录/
└── offline-tools/
    └── {工具ID}/
        └── {工具包文件名}.{扩展名}
```

**说明：**
- 「工具箱软件目录」指应用部署目录或用户数据目录
- 每个离线工具包按其 ID 单独存储在一个子目录中
- 导出时将文件转为 base64 嵌入 JSON；导入时还原为文件

## 9. 关键约束

| 约束项 | 内容 |
|--------|------|
| 数据隔离 | 数据存储在本地浏览器，不上传服务器 |
| 用户隔离 | 无用户系统，各浏览器数据独立 |
| 工具持久性 | 工具数据持久保存，直到用户主动删除 |
| 离线工具保护 | 离线工具包存储于工具箱软件目录下，防止用户误删 |
| 存储限制 | 单个图标不超过100KB，单个离线工具包不超过50MB |
| 数据迁移 | 通过导出导入JSON文件实现跨设备迁移 |
